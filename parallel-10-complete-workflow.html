Lesson 10 of 10 Advanced Track: Multi-Agent Orchestration

This is the capstone. You’ve learned every piece individually—sessions, agents, dashboards, synchronized commands, conflict resolution, scripts, and log capture. Now you’ll run the entire orchestration cycle from scratch in a single exercise, then wrap up with a quick-reference card for everything you’ve learned.

The Orchestration Cycle

Every multi-agent workflow follows the same six-phase cycle. This lesson walks through all six in sequence, applying them to a new feature enhancement:

Phase What you do Lesson
1. Launch Run the launcher script to create sessions and start agents 8
2. Assign Send each agent a clear task with ownership boundaries 3, 7
3. Monitor Watch progress from the dashboard and broadcast status checks 4, 6
4. Integrate Test both components together, fix mismatches 5
5. Capture Save session logs and run the summary 9
6. Clean up Kill agent sessions, keep control for continued work —

Final Exercise: End-to-End Run

1 Launch the environment

Start from a clean slate using the launcher script from Lesson 8:

~/projects/bookmarks/start-agents.sh

You should land in the control session with the dashboard layout ready and both agents running in the background.

2 Assign coordinated tasks

Give each agent an enhancement task. Claude adds a new API endpoint; Codex adds the matching CLI command:

# Give Claude an enhancement task tmux send-keys -t claude-api "Add a PATCH /bookmarks/:id endpoint to update bookmark title and tags" Enter # Give Codex a matching task (aligned with Claude's API contract) tmux send-keys -t codex-cli "Read ../handoff/api-contract.md, then add a 'bm edit <id> --title <new> --tags <new>' command that follows that contract" Enter
3 Monitor from the dashboard

Watch both agents’ progress in the dashboard’s top panes. If you set up the watch commands from Lesson 4, you’ll see live updates. Otherwise, do a manual check:

tmux capture-pane -t claude-api -p | tail -10 tmux capture-pane -t codex-cli -p | tail -10

Wait until both agents have finished before moving to the next step.

4 Test the integration

Start the API and run through the full CLI workflow, including the new edit command:

cd ~/projects/bookmarks/api && node index.js & cd ~/projects/bookmarks/cli node bm.js add https://test.com "Test" node bm.js list node bm.js edit 1 --title "Updated Title" node bm.js list

If the edit command works and the updated title appears in the list, the integration is complete. If not, use the identify → fix → retest cycle from Lesson 5.

5 Capture session logs

Save the session histories and run the summary:

cd ~/projects/bookmarks tmux capture-pane -t claude-api -p -S -1000 > claude-session.log tmux capture-pane -t codex-cli -p -S -1000 > codex-session.log ./summarize-session.sh
6 Clean up

Kill the agent sessions but keep the control session for continued work:

tmux kill-session -t claude-api tmux kill-session -t codex-cli # Keep control session for continued work

Quick Reference

Session Management

Command Action
tmux new-session -d -s name Create a detached session
tmux ls List all sessions
tmux attach -t name Attach to a session
Ctrl+J s Session switcher
tmux kill-session -t name Kill a session

Remote Commands

Command Action
tmux send-keys -t session 'cmd' Enter Send a command to a session
tmux capture-pane -t session -p Get current session output
tmux capture-pane -t session -p -S -1000 Get full scrollback history

Multi-Agent Patterns

Pattern Use Case
Ownership boundaries Each agent owns specific directories
Shared config Human owns shared files; agents read only
Integration testing Human tests in the control session
Session scripts Automate setup with bash scripts

Where to Go from Here

You’ve completed the Multi-Agent Orchestration track. You can now coordinate Claude and Codex across independent tmux sessions with a handoff workflow. Some ideas for going further:

  • Add a third agent for testing and QA
  • Create specialized agents — one for docs, one for code, one for deployment
  • Build a conductor script that assigns tasks based on file types or git diffs
  • Experiment with tmux hooks to auto-capture logs on detach
