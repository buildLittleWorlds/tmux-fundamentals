Windows (Multiple Workspaces)

Lesson 6 of 10 Beginner Track: tmux + Claude Code

So far you’ve been working in panes—splits within a single screen. Windows take this further by giving you entirely separate workspaces within one session, like browser tabs for your terminal. You’ll use them to keep Claude Code in one window and git operations in another, switching instantly with a keystroke.

What Are Windows?

Windows are like browser tabs—separate workspaces within one tmux session. Each window can have its own arrangement of panes. While panes divide a single screen, windows give you entirely new screens to switch between.

Look at the tmux status bar at the bottom of your terminal. You’ll see something like 0:bash*—that’s your current (and only) window. The asterisk marks the active window.

Panes vs. Windows: Panes split your current view into visible sections. Windows are full-screen workspaces you switch between. Think of it this way: panes are like tiles on one desk, windows are like separate desks you can walk to.

Exercise

1 Check your current window

You’re currently in window 0. Look at the status bar at the bottom—you should see 0:bash*. The * marks the active window.

2 Create a new window

Press:

Ctrl+J c

You’re now in window 1. The status bar shows 1:bash*. Notice this is a completely fresh workspace—a clean shell with no panes.

3 Use this window for git

In this new window, set up a git repository for your td project:

cd ~/projects/td git init git add -A git commit -m "Initial td CLI"

(Don’t worry if git isn’t set up or the project doesn’t exist yet—the point is having a separate workspace.)

4 Create a third window

Press Ctrl+J c again. Now you have 3 windows. Look at the status bar:

0:bash 1:bash 2:bash*

The asterisk shows you’re in window 2. All three are listed.

5 Switch between windows

Try each of these navigation commands:

# Next window Ctrl+J n # Previous window Ctrl+J p # Jump directly to window 0 Ctrl+J 0 # Jump directly to window 1 Ctrl+J 1

Direct jumps with Ctrl+J <number> are the fastest way to navigate when you know which window you want.

6 Return to your Claude window

Go back to window 0 where your Claude Code panes are:

Ctrl+J 0

Your two-pane supervisor layout is exactly as you left it. Each window preserves its own pane arrangement.

Checkpoint

  • The status bar shows multiple windows (numbered)
  • You can jump between windows with Ctrl+J <number>
  • Each window is independent with its own shell and pane layout

Organizing Your Workspace

A typical development setup uses windows to separate concerns:

Window Purpose
Window 0 Claude Code + testing pane (supervisor workflow)
Window 1 Git operations
Window 2 Documentation / notes
Window 3 Server logs

Name your windows for clarity by pressing Ctrl+J , and typing a descriptive name. The status bar will show the name instead of bash.

Window Commands Reference

Command Action
Ctrl+J c Create new window
Ctrl+J n Next window
Ctrl+J p Previous window
Ctrl+J 0–9 Jump to window by number
Ctrl+J , Rename current window
Ctrl+J & Close current window
