Putting It Together

Lesson 10 of 10 Beginner Track: tmux + Claude Code

This final lesson brings everything together. You’ll set up a complete multi-window workspace, give Claude one last round of polish requests, test every feature of the td CLI, and commit your finished project—using every tmux skill you’ve learned across the previous nine lessons.

The Complete Workflow

Over the course of this track you’ve built up a layered set of skills:

  • Sessions — persistent workspaces that survive disconnects
  • Panes — side-by-side views for the supervisor workflow
  • Windows — separate desks for different concerns
  • Backgrounding — keep working while Claude thinks
  • Headless sessions — fire-and-forget agents
  • Copy mode — scroll back, search, and copy output

Now you’ll use them all at once to finish the td tool.

Final Exercise: Complete the Tool

1 Attach to your tutorial session

If you’re not already inside tmux, reattach:

tmux attach -t tutorial
2 Set up your workspace

Arrange two windows for the final push:

  • Window 0: Claude + testing panes (split with Ctrl+J %)
  • Window 1: Git operations (create with Ctrl+J c)

Navigate to window 0 (Ctrl+J 0) and make sure Claude Code is running in the right pane.

3 Request final polish from Claude

In the Claude pane (right), ask for a final round of improvements:

Review td.sh and make these improvements: 1. Add color output (green for success, red for errors) 2. Show task count in the list header 3. Add --version flag that shows "td v1.0" Make sure all existing features still work.

Watch Claude work, or background the task with Ctrl+B and check back with /status.

4 Test thoroughly

Switch to the left pane (Ctrl+J ←) and run through every feature:

./td.sh --version ./td.sh --help ./td.sh clear ./td.sh add "First task" ./td.sh add "Second task" ./td.sh add "Third task" ./td.sh list ./td.sh done 2 ./td.sh list ./td.sh search "First" ./td.sh clear

If anything fails, switch back to Claude and describe the problem. This is the supervisor workflow in action—prompt, test, fix, repeat.

5 Commit your work in the git window

Switch to window 1:

Ctrl+J 1

Then commit the finished project:

cd ~/projects/td git add -A git commit -m "Complete td CLI tool"
6 (Optional) Install td globally

Make td available from any directory:

sudo cp td.sh /usr/local/bin/td # Now you can run "td" from anywhere

Checkpoint

Your td tool should support all of these commands:

  • td --help — show usage
  • td --version — show version
  • td add "task" — add a task
  • td list — show all tasks with count
  • td done <n> — remove task by number
  • td search <term> — find matching tasks
  • td clear — remove all tasks

tmux Quick Reference Card

Here’s a consolidated reference of every tmux command you’ve learned across this track.

Session Management

Command Action
tmux new -s name New named session
tmux attach -t name Attach to session
tmux ls List sessions
tmux kill-session -t name Kill session
tmux new-session -d -s name "cmd" Start headless session
Ctrl+J d Detach from session

Panes

Command Action
Ctrl+J % Split vertically (left/right)
Ctrl+J " Split horizontally (top/bottom)
Ctrl+J ↑↓←→ Move between panes
Ctrl+J x Close current pane
Ctrl+J z Zoom pane (toggle fullscreen)

Windows

Command Action
Ctrl+J c New window
Ctrl+J n / p Next / previous window
Ctrl+J 0–9 Jump to window by number
Ctrl+J , Rename window
Ctrl+J & Close window

Copy Mode

Command Action
Ctrl+J [ Enter copy mode
↑ / ↓ / PgUp / PgDn Navigate
/ pattern Search forward
? pattern Search backward
n / N Next / previous match
q Exit copy mode

Claude Code in tmux

Command Action
Ctrl+J (alone) Background current task
/status Check background task
/resume Resume background task
