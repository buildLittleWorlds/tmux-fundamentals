Launch Claude Code

Lesson 4 of 10 Beginner Track: tmux + Claude Code

Now you’ll combine what you’ve learned—sessions, panes, and navigation—into a real workflow. You’ll start Claude Code in one pane, give it a task, and verify the results in the other pane. This is the first taste of the supervisor pattern you’ll use for the rest of the track.

Claude Code in a Pane

Claude Code is a CLI tool that generates and edits code from natural-language prompts. Running it inside a tmux pane means you can watch its output in real time while keeping a separate shell available for testing, browsing files, and running git commands.

In this lesson you’ll create a small project—a bash todo CLI called td—and use Claude Code to generate the initial scaffolding.

Exercise

1 Start with a single pane

If you still have multiple panes open from Lesson 3, close the extras with exit until you’re down to one pane.

2 Create a project directory

Set up a workspace for the td tool:

mkdir -p ~/projects/td && cd ~/projects/td
3 Split into two panes

Split vertically so you have a left pane and a right pane:

Ctrl+J %

Your cursor lands in the right pane.

4 Launch Claude Code in the right pane

Start Claude Code:

claude

Wait for the Claude Code prompt to appear. The right pane is now your AI workspace.

5 Give Claude its first task

Type the following prompt into Claude Code:

Create a simple CLI todo app called "td" in a single file called td.sh. For now, just make it show help text when run with --help or no arguments. Keep it minimal - just bash, no dependencies.

Watch as Claude generates the file. You can see every line it writes in real time.

6 Test in the left pane

Move to the left pane (Ctrl+J ←) and verify what Claude created:

chmod +x td.sh ./td.sh --help

You should see usage information printed to the terminal.

Checkpoint

  • Claude Code is running in the right pane
  • A file td.sh exists in your project directory
  • Running ./td.sh --help shows usage information

The Supervisor Pattern

Notice what just happened: Claude worked in one pane while you verified the results in another. You could watch Claude’s progress in real time and test immediately without switching windows or waiting. This is the supervisor workflow—you’ll use it constantly throughout this track.

Left pane = you, right pane = Claude. This convention keeps things predictable. You always know where to look for Claude’s output and where to run your own commands. In the next lesson, you’ll formalize this pattern and start iterating on the td tool with rapid prompt–test cycles.
