Detach and Reattach

Lesson 2 of 10 Beginner Track: tmux + Codex CLI

The killer feature of tmux is detaching—you can disconnect from a session and reconnect later with everything still running. This is crucial when waiting for Codex to generate long code blocks: detach, do something else, and reattach when you’re ready.

Why Detaching Matters

When you detach from a tmux session, you return to your regular terminal—but the session keeps running in the background. Every command, every process, every bit of output continues exactly as if you were still watching. When you reattach, you pick up right where you left off.

This is the foundation for working with AI CLI tools. Codex can take time to generate complex code. Instead of staring at a spinner, you detach, work on something else, and come back when the output is ready.

What you’ll do: Start a long-running command, detach from the session, verify it’s still alive, and reattach to see that the output continued while you were away.

Exercise

1 Make sure you’re inside tmux

You should be in the codex-lab session from Lesson 1. If you’re not, reattach:

tmux attach -t codex-lab
2 Start a command that shows activity

Run a loop that prints a message every two seconds. This gives you something visible to come back to:

while true; do echo "Waiting for Codex..."; sleep 2; done

You should see “Waiting for Codex...” appearing repeatedly.

3 Detach from the session

Press the prefix key followed by d:

Ctrl+J d

You’re back at your regular terminal. The green status bar is gone. The loop is still running inside the detached session—you just can’t see it.

4 Verify the session is still alive

List all tmux sessions:

tmux list-sessions

You should see your codex-lab session listed. The output confirms it’s still running even though you’re no longer attached to it.

5 Reattach to the session

Reconnect to the session by name:

tmux attach -t codex-lab

You’re back inside tmux. Notice the loop kept printing “Waiting for Codex...” the entire time you were detached. The status bar is back.

6 Stop the loop

Press Ctrl+c to stop the loop. You’re back to a normal shell prompt inside your tmux session, ready for the next lesson.

Checkpoint

  • After detaching, tmux ls shows your session is still running
  • After reattaching, the output continued while you were “away”
  • Your session survived the full detach/attach cycle
