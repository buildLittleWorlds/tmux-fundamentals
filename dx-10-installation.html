From Project Directory to System Command

Right now you run dx by calling ./dx or ~/projects/dx/dx directly. To use it from any directory, you need it in your PATH. The install script creates a thin wrapper in ~/.local/bin that forwards all arguments to the real dx script—so you can update the source in place and the installed command always uses the latest version.

Exercise: Add Version Info and Install

1 Add a version command

This simple subcommand prints the version and lists all available commands by reading the commands/ directory:

~/projects/dx/commands/version
#!/bin/bash
echo "dx version 1.0.0"
echo "AI backends: claude, codex"
echo ""
echo "Commands available:"
ls -1 "$(dirname "$0")" | grep -v "^_"

Make it executable:

chmod +x commands/version

The ls -1 lists all files in the commands/ directory, and grep -v "^_" filters out any helper files prefixed with an underscore (a convention for non-command files if you add them later).

2 Create an install script

The installer creates a wrapper script in ~/.local/bin that calls the real dx from its source directory. This way updates to the source are immediately available everywhere:

~/projects/dx/install.sh
#!/bin/bash
set -e

INSTALL_DIR="${1:-$HOME/.local/bin}"
SOURCE_DIR="$(cd "$(dirname "$0")" && pwd)"

mkdir -p "$INSTALL_DIR"

# Create wrapper that calls the real dx
cat > "$INSTALL_DIR/dx" << WRAPPER
#!/bin/bash
exec "$SOURCE_DIR/dx" "\$@"
WRAPPER
chmod +x "$INSTALL_DIR/dx"

echo "Installed dx to $INSTALL_DIR/dx"
echo ""
echo "Make sure $INSTALL_DIR is in your PATH:"
echo "  export PATH=\"$INSTALL_DIR:\$PATH\""
echo ""
echo "Then run: dx help"

Make it executable:

chmod +x install.sh

Key details:

  • Custom install path — Pass a directory as the first argument to install elsewhere (e.g., ./install.sh /usr/local/bin). Defaults to ~/.local/bin.
  • exec — The wrapper uses exec to replace itself with the real dx process, so there’s no extra shell sitting in the process tree.
  • Escaped \$@ — The WRAPPER heredoc uses a backslash to preserve the literal $@ in the generated file, ensuring all arguments are forwarded correctly.
3 Install dx

Run the installer, add the directory to your PATH, and verify:

cd ~/projects/dx ./install.sh export PATH="$HOME/.local/bin:$PATH" dx help

You should see the full help output. Now dx works from any directory.

4 Make it permanent

Add the PATH export to your shell profile so it persists across terminal sessions:

# For zsh (default on macOS) echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc # For bash echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc

Open a new terminal and confirm dx help works without any manual export.

The Complete dx Toolkit

Here’s every command you’ve built across this track:

dx help # Show all commands dx version # Show version info dx review # Review staged changes dx pr # Create PR with AI description dx explain <file> # Interactive code explanation dx test # Run tests, analyze failures dx standup # Summarize git activity dx ask "question" # Quick popup Q&A dx watch "*.py" # Watch files for changes

All commands accept --ai=claude or --ai=codex to switch AI backends.

Quick Reference: tmux Integration Patterns

These are the five tmux patterns you used to build dx. Each maps to a different kind of AI interaction:

Pattern Use Case
ai_prompt "$prompt" One-shot, non-interactive
ai_interactive "$prompt" Persistent session with context
tmux split-window -h "$AI_CLI" Side-by-side analysis
tmux display-popup -E "$AI_CLI -p '$prompt'" Floating popup
tmux new-session -d -s name "command" Background watcher
