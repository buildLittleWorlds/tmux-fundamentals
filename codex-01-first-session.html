Your First Session

Lesson 1 of 10 Beginner Track: tmux + Codex CLI

Everything in tmux starts with a session—a persistent workspace that keeps running even if you close your terminal. In this lesson you’ll create your first named session, run a few commands inside it, and learn to read the tmux status bar that tells you where you are.

What Is a tmux Session?

A tmux session is a persistent workspace. Programs running inside it continue even if you close your terminal window or disconnect from a remote server. This is what makes tmux invaluable when working with AI tools like Codex CLI—long-running code generation tasks keep going whether you’re watching or not.

What you’ll do: Create a named tmux session called codex-lab, run some commands inside it, and identify the key parts of the tmux status bar.

Exercise

1 Open your terminal

Launch your regular terminal emulator. You should be at a normal shell prompt—no tmux running yet.

2 Start a named tmux session

Create a new session called codex-lab:

tmux new -s codex-lab

The -s flag gives the session a name. Named sessions are easier to find and reattach to later.

3 Run some commands inside tmux

Your terminal looks different now—notice the status bar at the bottom. Try a few commands to confirm everything works normally:

echo "Hello from inside tmux!" pwd ls

Your shell works exactly the same inside tmux. The only visible difference is the green status bar.

4 Read the status bar

Look at the bottom of your screen. The tmux status bar shows:

  • Left side: [codex-lab] — your session name
  • Center: 0:bash* — window number and name (the * marks the active window)
  • Right side: hostname and current time

This bar is your dashboard. As you add windows and panes in later lessons, it will show you everything at a glance.

Checkpoint

  • A status bar is visible at the bottom of your terminal (usually green)
  • The left side reads [codex-lab] — your session name
  • The right side shows your hostname and the current time
  • Your shell prompt works normally inside the session

What’s Next: You’ve created a tmux session—but the real magic is that it persists. In Lesson 2: Detach and Reattach, you’ll learn to disconnect from a session and reconnect later with everything still running.
