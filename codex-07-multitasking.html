Multitasking with Codex

Lesson 7 of 10 Beginner Track: tmux + Codex CLI

Codex doesn’t need you to sit and watch while it works. With tmux, you can switch to another pane and keep working—editing files, running commands, or browsing logs—while Codex generates code in the background. This lesson also introduces Codex’s built-in commands for managing your session.

Work While Codex Thinks

When you give Codex a complex prompt, it can take a while to generate a full response. In a traditional terminal, you’d be stuck watching output scroll. With tmux panes, you simply switch away and do something else.

The key insight: Codex keeps running in its pane regardless of which pane you’re looking at. You can glance over at any time to check progress—when the text stops scrolling, the output is ready.

Codex Built-in Commands

Codex CLI has slash commands that help you manage your session without leaving the conversation:

Command What It Does
/status Show current session settings and context
/compact Summarize conversation history to save context window space

What you’ll build: Three new features for td—a done command to remove completed tasks, a clear command to wipe the list, and better error handling—all while practicing the multitasking workflow.

Exercise

1 Navigate to your Codex pane

Make sure you’re in window 0 with your two-pane layout. Switch to the Codex pane (right side).

2 Give Codex a bigger task

This prompt asks for multiple features at once—it will take Codex a moment to generate everything:

Refactor td.sh to include: 1. A "done" command: ./td.sh done 2 (marks task 2 as complete by removing it) 2. A "clear" command: ./td.sh clear (removes all tasks) 3. Better error handling for invalid commands
3 Immediately switch panes

Don’t wait for Codex to finish. As soon as you send the prompt, switch to the left pane:

Ctrl+J ←

This is the key habit: prompt, then move. Let Codex work while you do something else.

4 Work while Codex generates

While Codex is “thinking” and printing output in the right pane, you’re free to do anything in the left pane—run ls, check top, edit files, or review git history. The panes are completely independent.

Tip: Glance at the right pane periodically. When the text stops scrolling, Codex is done and the new code is ready to test.

5 Test the new features

Once Codex has finished, test the changes in the left pane:

./td.sh list ./td.sh done 1 ./td.sh list ./td.sh clear

The done command should remove a task by number, and clear should wipe all tasks. If something doesn’t work, switch back to Codex and describe the problem.

6 Try Codex built-in commands

Switch back to the Codex pane (Ctrl+J →) and try these slash commands:

/status /compact

/status shows your current session settings and context. /compact summarizes the conversation history to free up context window space—useful after long sessions with many prompts.

Checkpoint

  • You can move freely between panes while Codex is generating output
  • The done and clear commands work in td.sh
  • /status shows current session status
  • /compact helps trim long conversations

What’s Next: When Codex produces a lot of output, you’ll need to scroll back through it. In Lesson 8: Copy Mode, you’ll learn tmux’s built-in scroll and search feature for handling long output.
