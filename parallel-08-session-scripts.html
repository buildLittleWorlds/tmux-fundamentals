Session Scripts

Lesson 8 of 10 Advanced Track: Multi-Agent Orchestration

Over the past seven lessons you’ve been typing the same setup commands every time: create sessions, launch agents, split panes, arrange the dashboard. It’s time to capture all of that into a single launcher script so your entire multi-agent environment spins up with one command.

Why Script Your Sessions

Every tmux command you’ve used—new-session, send-keys, split-window—is just a shell command. That means your entire multi-agent setup is scriptable. A launcher script gives you:

  • Reproducibility — the same environment every time, no forgotten steps
  • Speed — one command instead of a dozen
  • Shareability — teammates can use the same script to get the same layout

This is the tmux power pattern. Experienced tmux users almost always have launcher scripts for their projects. What you’re building here is a standard practice—just applied to AI agent orchestration.

Exercise: Create the Launcher Script

1 Write the script

Create a bash script that sets up the full multi-agent environment:

~/projects/bookmarks/start-agents.sh
#!/bin/bash
PROJECT_DIR=~/projects/bookmarks

# Kill existing sessions if they exist
tmux kill-session -t claude-api 2>/dev/null
tmux kill-session -t codex-cli 2>/dev/null
tmux kill-session -t control 2>/dev/null

# Create sessions
tmux new-session -d -s claude-api -c "$PROJECT_DIR/api"
tmux new-session -d -s codex-cli -c "$PROJECT_DIR/cli"
tmux new-session -d -s control -c "$PROJECT_DIR"
mkdir -p "$PROJECT_DIR/handoff"

# Start the AI agents
tmux send-keys -t claude-api 'claude' Enter
tmux send-keys -t codex-cli 'codex --no-alt-screen' Enter

# Set up control session layout
tmux send-keys -t control 'tmux split-window -v -p 30' Enter
tmux send-keys -t control 'tmux select-pane -t 0' Enter
tmux send-keys -t control 'tmux split-window -h' Enter

# Attach to control
tmux attach -t control

Then make it executable:

chmod +x ~/projects/bookmarks/start-agents.sh

What the script does

Walking through the script section by section:

Section Purpose
kill-session lines Clean up any leftover sessions from a previous run. The 2>/dev/null suppresses errors if the session doesn’t exist.
new-session lines Create three detached sessions, each with its own working directory (-c).
send-keys for agents Launch Claude and Codex inside their respective sessions.
send-keys for layout Set up the control session’s dashboard layout (split panes from Lesson 4).
tmux attach Drop you into the control session, ready to orchestrate.

Exercise: Test the Script

2 Kill existing sessions and run the launcher

Start from a clean slate and test the script end to end:

# Kill all tmux sessions tmux kill-server # Run the launcher ~/projects/bookmarks/start-agents.sh

You should land in the control session with the dashboard layout already arranged. Both agent sessions should be running in the background with Claude and Codex started.

3 Verify everything is running

Confirm all three sessions are active and the agents are responding:

# List all sessions tmux ls # Quick peek at each agent tmux capture-pane -t claude-api -p | tail -5 tmux capture-pane -t codex-cli -p | tail -5

You should see three sessions listed and recent output from both agents confirming they’re ready for input.

Note: tmux kill-server destroys all tmux sessions, not just the ones from this project. If you have other tmux work running, use the individual kill-session commands from the script instead.

Checkpoint

  • One command sets up your entire multi-agent environment
  • Sessions are created with the correct working directories
  • AI agents start automatically in their respective sessions
