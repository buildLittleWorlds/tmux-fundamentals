The PR Command

Lesson 4 of 10 Advanced Track: Developer Toolkit

dx pr takes the pattern from dx review a step further—it generates a PR title and description from your commits using AI, shows you a preview, and then creates the pull request with gh. This is your first command that combines AI output with a real shell action.

AI Output as Input to Shell Commands

The review command printed AI output and stopped. The PR command parses the AI’s response and feeds it into another tool (gh pr create). This introduces a new challenge: you need the AI to respond in a predictable, parseable format.

The trick is a structured prompt with explicit format instructions and delimiters (TITLE:, BODY:, END_BODY). The script then uses grep and sed to extract each section. If parsing fails, it falls back to showing the raw output so you can copy-paste manually.

Prerequisite: This command requires the GitHub CLI (gh). Install it with brew install gh and authenticate with gh auth login before testing.

Exercise: Build the PR Command

1 Create the pr command

This script detects the base branch, gathers commit history and diff stats, asks the AI for a title and body, then offers to create the PR:

~/projects/dx/commands/pr
#!/bin/bash
set -e

SCRIPT_DIR="$(cd "$(dirname "$0")/.." && pwd)"
source "$SCRIPT_DIR/lib.sh"
parse_ai_flag "$@"

# Detect base branch
BASE_BRANCH="${BASE_BRANCH:-main}"
if ! git rev-parse --verify "$BASE_BRANCH" >/dev/null 2>&1; then
  BASE_BRANCH="master"
fi

# Get commits not in base branch
COMMITS=$(git log "$BASE_BRANCH"..HEAD --oneline 2>/dev/null || echo "")
DIFF_STAT=$(git diff "$BASE_BRANCH"..HEAD --stat 2>/dev/null || echo "")

if [[ -z "$COMMITS" ]]; then
  echo "No commits to include in PR."
  echo "Make sure you're on a feature branch with commits ahead of $BASE_BRANCH"
  exit 1
fi

echo "Generating PR description with $AI_CLI..."

# Generate PR content
PROMPT="Generate a GitHub PR title and description based on these commits:

Commits:
$COMMITS

Files changed:
$DIFF_STAT

Format your response EXACTLY like this (I will parse it):
TITLE: <short title under 60 chars>
BODY:
<markdown body with:
- Summary (1-2 sentences)
- Changes (bullet points)
- Testing notes (if applicable)>
END_BODY"

RESPONSE=$(ai_prompt "$PROMPT")

# Parse the response
TITLE=$(echo "$RESPONSE" | grep "^TITLE:" | sed 's/^TITLE: //')
BODY=$(echo "$RESPONSE" | sed -n '/^BODY:/,/^END_BODY/p' | sed '1d;$d')

if [[ -z "$TITLE" ]]; then
  echo "Failed to parse AI response. Raw output:"
  echo "$RESPONSE"
  exit 1
fi

echo ""
echo "Generated PR:"
echo "============="
echo "Title: $TITLE"
echo ""
echo "Body:"
echo "$BODY"
echo ""
echo "============="
echo ""

read -p "Create this PR? [y/N] " -n 1 -r
echo ""

if [[ $REPLY =~ ^[Yy]$ ]]; then
  gh pr create --title "$TITLE" --body "$BODY"
else
  echo "PR creation cancelled."
fi

Make it executable:

chmod +x commands/pr

Walk through the key sections:

  • Base branch detection — Defaults to main, falls back to master if it doesn’t exist. You can override with BASE_BRANCH=develop dx pr.
  • Commit gathering — git log main..HEAD --oneline gets only the commits on your feature branch. git diff --stat shows which files changed and how much.
  • Structured prompt — The format instructions with TITLE:, BODY:, and END_BODY delimiters make the AI’s response machine-parseable.
  • Confirmation step — The read -p prompt lets you review before the PR is actually created. Saying no cancels cleanly.
2 Install the GitHub CLI (if needed)

The gh CLI handles authentication and PR creation. If you don’t have it yet:

# macOS brew install gh gh auth login

Follow the interactive prompts to authenticate with your GitHub account. You only need to do this once.

Your project structure now looks like this:

~/projects/dx/ ├── dx # Main dispatcher script ├── lib.sh # Shared AI functions └── commands/ ├── review # AI-powered code review └── pr # AI-generated pull requests

Checkpoint

  • dx pr generates a title and description from your branch’s commits
  • A preview is shown before the PR is created
  • Confirming with y uses gh to create the pull request
