Windows (Multiple Workspaces)

Lesson 6 of 10 Beginner Track: tmux + Codex CLI

So far you’ve been working inside a single window with panes. Windows are like browser tabs for your terminal—separate workspaces within one session that let you keep different tasks organized. In this lesson you’ll create a dedicated git window alongside your Codex workspace.

Windows vs. Panes

Panes split a single screen so you can see multiple things at once. Windows give you entirely separate screens you can flip between—like tabs in a browser. Each window can have its own pane layout.

This is useful when you want a clean, full-screen workspace for a task (like git operations) without disturbing your Codex + testing layout.

Key Action
Ctrl+J c Create a new window
Ctrl+J n Next window
Ctrl+J p Previous window
Ctrl+J 0 Jump to window 0
Ctrl+J 1 Jump to window 1

What you’ll build: A second window dedicated to git, where you’ll initialize a repository for the td project you’ve been building with Codex.

Exercise

1 Check your current window

Look at the bottom status bar. You should see something like 0:bash*—this means you’re in window 0. The asterisk marks the active window.

2 Create a new window

Press Ctrl+J c to create a new window. Your screen will switch to a fresh shell. The status bar now shows two windows: 0:bash and 1:bash*.

Tip: Your pane layout in window 0 (with Codex running) is preserved. Creating a new window doesn’t disturb existing windows.

3 Initialize a git repository

In this new window, set up version control for the td project:

cd ~/projects/td git init git add -A git commit -m "Initial td CLI"

Now you have a dedicated git workspace without cluttering your Codex panes.

4 Switch between windows

Practice moving between your windows:

# Next window Ctrl+J n # Previous window Ctrl+J p

Each time you switch, notice how the status bar updates the asterisk to show which window is active.

5 Jump directly to a window

Go back to window 0 where Codex is running:

Ctrl+J 0

You can jump to any window by number. This is faster than cycling through with n and p when you have several windows.

Checkpoint

  • Your status bar shows multiple windows (numbered)
  • You can create a new window with Ctrl+J c
  • You can cycle windows with Ctrl+J n and Ctrl+J p
  • You can jump to any window with Ctrl+J <number>
  • Your Codex panes in window 0 are still intact

What’s Next: Now that you can organize work across windows, in Lesson 7: Multitasking you’ll combine panes and windows to run Codex tasks in parallel—multiple AI agents working simultaneously while you supervise from a central window.
