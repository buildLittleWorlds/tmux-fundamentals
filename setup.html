Setup & Configuration

Before starting any track, configure your tmux prefix key, terminal settings, and AI CLI tools. This page is a shared reference — bookmark it and come back whenever you need to check a setting or troubleshoot an issue.

Ctrl+J Prefix Setup

All tutorials in this curriculum use Ctrl+J as the tmux prefix key instead of the default Ctrl+B. This avoids conflicts with common shortcuts in AI CLIs and keeps your hands on the home row.

How the Prefix Key Works

tmux commands are two-step: press the prefix, release, then press the action key. For example:

  1. Press Ctrl+J (hold Ctrl, tap j, release both)
  2. Then press the action key (e.g., d to detach, % to split)

When you see Ctrl+J d in a tutorial, that means: press Ctrl+J, release, then press d.

Configure tmux

Add the following to your ~/.tmux.conf file (create it if it doesn’t exist):

~/.tmux.conf
# Use Ctrl+J as the tmux prefix
unbind C-b
unbind C-j
unbind C-a
set-option -g prefix C-j
bind-key C-j send-prefix

Reload the config without restarting tmux:

$ tmux source ~/.tmux.conf

Recommended tmux Config

These settings improve the experience when working with AI CLIs that produce long, colorful output:

~/.tmux.conf (additional settings)
# True color support (for syntax highlighting)
set -g default-terminal "xterm-256color"
set-option -ga terminal-overrides ",xterm-256color:Tc"

# Large scrollback (AI responses can be long)
set -g history-limit 50000

# Mouse support (easier scrolling)
set -g mouse on

# Start window numbering at 1
set -g base-index 1

# Better split keys (optional)
bind | split-window -h
bind - split-window -v

Claude Code Keybinding Remap (Optional)

Claude Code uses Ctrl+B by default to background the current task. Since we’re using Ctrl+J as the tmux prefix (instead of the default Ctrl+B), this avoids any conflict with Claude’s background-task key. If you still want a different background key, you can remap it:

~/.claude/keybindings.json
{
  "bindings": [
    { "context": "chat", "keys": { "ctrl+x": "background_task" } }
  ]
}

Note: This remap is optional. Since we changed the tmux prefix to Ctrl+J, Ctrl+B is free for Claude Code’s background feature and works without any conflict.

Codex CLI: No-Alt-Screen Mode

Codex CLI uses an alternate screen buffer by default, which can interfere with tmux scrollback. Launch Codex with the --no-alt-screen flag for better compatibility:

$ codex --no-alt-screen

This keeps Codex output in the normal scrollback buffer so you can use tmux copy mode (Ctrl+J [) to review previous output.

Troubleshooting

Issue Solution
Shift+Enter submits instead of newline Use \ at end of line, or Ctrl+X to open an external editor
Can’t copy text across pane borders Hold Option (Mac) or Shift (Linux) while selecting
Colors look washed out Ensure terminal-overrides is set in tmux.conf (see above)
Codex scrollback is hard to navigate Launch Codex with codex --no-alt-screen
Prefix key doesn’t work Run tmux source ~/.tmux.conf or restart tmux
tmux not installed brew install tmux (macOS) or sudo apt install tmux (Linux)

Quick Reference Card

Session Management

Command Action
tmux new -s name New named session
tmux attach -t name Attach to session
tmux ls List sessions
tmux kill-session -t name Kill session
Ctrl+J d Detach from session
Ctrl+J s Session switcher

Panes

Command Action
Ctrl+J % Split vertically (left/right)
Ctrl+J " Split horizontally (top/bottom)
Ctrl+J ↑↓←→ Move between panes
Ctrl+J x Close current pane
Ctrl+J z Zoom pane (toggle fullscreen)

Windows

Command Action
Ctrl+J c New window
Ctrl+J n / Ctrl+J p Next / previous window
Ctrl+J 0-9 Jump to window by number
Ctrl+J , Rename window
Ctrl+J & Close window

Copy Mode

Command Action
Ctrl+J [ Enter copy mode
↑↓ / PgUp / PgDn Navigate
Ctrl+S / Ctrl+R Search forward / backward
n / N Next / previous match
Ctrl+Space Start selection
Alt+W Copy selection
Ctrl+J ] Paste
q Exit copy mode

Remote Commands (Advanced)

Command Action
tmux send-keys -t session 'cmd' Enter Send command to a session
tmux capture-pane -t session -p Get session output
tmux new-session -d -s name "cmd" Detached session with command
