Headless Codex Sessions

Lesson 9 of 10 Beginner Track: tmux + Codex CLI

Sometimes you want to fire off a Codex task and walk away—close your laptop, grab coffee, come back later. By combining tmux’s detached sessions with Codex’s non-interactive mode, you can run AI-powered tasks in the background without keeping a terminal open.

Two Modes of Headless Work

There are two ways to run Codex without babysitting it:

Approach How It Works Best For
codex exec Runs a single prompt non-interactively and exits One-shot tasks, scripting
tmux new-session -d Starts a detached session running any command Long-running tasks you want to check on later

The real power comes from combining them: launch a detached tmux session that runs codex exec, and you have a background AI agent that works while you’re away.

What you’ll build: A search command for td—built entirely by a headless Codex agent running in a detached tmux session.

Exercise

1 Detach from your current session

First, leave your existing session running in the background:

Ctrl+J d

You’re back at your regular shell. Your tmux session (with Codex and all your panes) is still running—you just can’t see it.

2 Try codex exec directly

The exec subcommand runs Codex non-interactively—it processes a single prompt and exits:

cd ~/projects/td codex exec "Explain what td.sh does in one paragraph"

Codex prints its response and returns you to the shell. No interactive session, no follow-up—just a one-shot answer.

3 Launch a detached Codex agent

Now combine both techniques. Start a new tmux session in the background that runs a Codex task:

tmux new-session -d -s agent "cd ~/projects/td && codex exec \"Add a search command to td.sh: ./td.sh search <term> shows only matching tasks\""

This creates a session named agent that runs detached. Codex is working in the background—you don’t need to watch.

4 List your sessions

Verify both sessions are running:

tmux ls

You should see both your original session (codex-lab) and the new agent session.

5 Check on the agent

Attach to the agent session to see how it’s doing:

tmux attach -t agent

If Codex is still running, you’ll see output scrolling. If it’s finished, you’ll see the completed output and a shell prompt.

6 Test the new search command

Once Codex has finished, detach (Ctrl+J d) and test the new feature:

cd ~/projects/td ./td.sh add "Learn Codex CLI" ./td.sh add "Master tmux" ./td.sh search "Codex"

The search command should filter your task list and show only matching entries.

7 Clean up the agent session

Once you’ve confirmed the feature works, remove the temporary session:

tmux kill-session -t agent

Then reattach to your main session to continue working:

tmux attach -t codex-lab

Checkpoint

  • codex exec "prompt" runs Codex non-interactively
  • tmux new-session -d -s name starts a detached session
  • You can check on a detached session with tmux attach -t name
  • Codex can perform tasks in the background without your terminal open
  • The search command works in td.sh

What’s Next: You’ve learned all the core tmux + Codex skills. In Lesson 10: Putting It Together, you’ll combine everything into a complete development workflow—panes, windows, detached sessions, and Codex—to build a polished final feature for td.
