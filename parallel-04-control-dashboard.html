The Control Dashboard

Lesson 4 of 10 Advanced Track: Multi-Agent Orchestration

Switching between sessions to check on agents works, but it’s slow. In this lesson you’ll turn your control session into a proper monitoring dashboard—a three-pane layout where you can see both agents’ output live while keeping a command center at the bottom.

The Dashboard Layout

The goal is a single window in your control session that shows everything at once:

┌─────────────────┬─────────────────────────┐ │ Claude tail │ Codex tail │ ├─────────────────┴─────────────────────────┤ │ Command center │ └───────────────────────────────────────────┘

The top half is split into two monitoring panes, each running a watch command that continuously captures output from a background session. The bottom pane is where you type commands—sending tasks to agents, checking files, running tests.

How it works

The watch command re-runs a shell command every N seconds and displays the result. Combined with tmux capture-pane, it creates a live feed of what’s happening in another session:

watch -n 2 'tmux capture-pane -t session-name -p | tail -15'

This refreshes every 2 seconds (-n 2) and shows the last 15 lines of the target session. It’s read-only—you’re watching, not interacting.

Exercise: Build the Dashboard

1 Reset the control session layout

Start with a clean slate by closing any existing panes, then build the three-pane layout:

# Kill all panes except the current one tmux kill-pane -a # Split: top 70% and bottom 30% tmux split-window -v -p 30 # Select the top pane and split it left/right tmux select-pane -t 0 tmux split-window -h

You now have three panes: top-left, top-right, and bottom.

2 Start the Claude monitor (top-left pane)

You should already be in the top-left pane after the split. Start watching Claude’s session:

# Continuously show Claude's recent output watch -n 2 'tmux capture-pane -t claude-api -p | tail -15'

You’ll see the last 15 lines of Claude’s session, refreshing every 2 seconds.

3 Start the Codex monitor (top-right pane)

Move to the top-right pane and start watching Codex:

# Navigate to top-right pane Ctrl+J →

Then run:

# Continuously show Codex's recent output watch -n 2 'tmux capture-pane -t codex-cli -p | tail -15'
4 Move to the command center (bottom pane)

Navigate to the bottom pane—this is your workspace for the rest of the track:

# Navigate to bottom pane Ctrl+J ↓

From here you can send commands to agents, check files, run tests, and manage the handoff—all while glancing up to see what each agent is doing.

Pane Layout Commands

Command What It Does
tmux kill-pane -a Close all panes except the current one
tmux split-window -v -p 30 Split horizontally, bottom pane gets 30% of height
tmux split-window -h Split vertically (left/right), equal width
tmux select-pane -t 0 Select pane by index number
watch -n 2 'command' Re-run a command every 2 seconds, display output

Tip: If the dashboard feels cramped, you can adjust the split ratio. Use -p 40 instead of -p 30 to give the command center more space, or resize panes interactively with Ctrl+J followed by holding an arrow key.

Checkpoint

  • Top-left pane shows Claude’s recent output, refreshing every 2 seconds
  • Top-right pane shows Codex’s recent output, refreshing every 2 seconds
  • Bottom pane is your command center where you type commands
  • You can see both agents’ progress at a glance without switching sessions
