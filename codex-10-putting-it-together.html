Putting It Together

Lesson 10 of 10 Beginner Track: tmux + Codex CLI

This is the capstone. You’ll combine every skill from this track—sessions, panes, windows, the supervisor workflow, multitasking, copy mode, and headless sessions—into a single polished workflow that adds the final features to td and commits the finished tool.

The Complete Workflow

Over the past nine lessons you’ve built up a layered toolkit:

Skill Lesson What It Gives You
Sessions 1 & 2 Persistent workspace that survives disconnects
Panes 3 Side-by-side views within a window
Codex integration 4 & 5 AI coding assistant in a dedicated pane
Windows 6 Separate workspaces (tabs) for different tasks
Multitasking 7 Work while Codex generates in the background
Copy mode 8 Scroll back and search through output
Headless sessions 9 Fire-and-forget background agents

Now you’ll use them all together in a realistic development session.

Final Exercise: Complete the Tool

1 Attach to your lab session

If you’re not already inside tmux, reattach to your workspace:

tmux attach -t codex-lab
2 Set up your workspace

Confirm you have a two-window layout:

  • Window 0 — Codex + testing panes (your supervisor workflow)
  • Window 1 — Git operations

If window 1 doesn’t exist yet, create it with Ctrl+J c and cd ~/projects/td. Jump back to window 0 with Ctrl+J 0.

3 Request final polish from Codex

In the Codex pane (right side of window 0), ask for the finishing touches:

Review td.sh and add color output (green for success, red for errors). Also add a --version flag.

While Codex works, switch to the left pane (Ctrl+J ←) and do something else—you know the drill.

4 Test thoroughly

Once Codex finishes, test the new features in the left pane:

./td.sh --version ./td.sh add "Final task" ./td.sh list

You should see colored output and a version string. If anything needs fixing, switch back to Codex and iterate—just like you’ve been doing since Lesson 5.

5 Commit your finished tool

Switch to window 1 (Ctrl+J 1) for git operations:

cd ~/projects/td git add -A git commit -m "Complete td CLI tool"

Your td tool is complete—built entirely through the tmux + Codex supervisor workflow.

Checkpoint

  • Your td tool has all features: add, list, done, clear, search, --version
  • Output uses color (green for success, red for errors)
  • Everything is committed to git
  • You used multiple windows, panes, and the supervisor workflow to build it

Track Complete! You’ve mastered the fundamentals of tmux with Codex CLI. You can create sessions, split panes, manage windows, multitask while Codex generates code, scroll through output with copy mode, and run headless AI agents.
