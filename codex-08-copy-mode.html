Copy Mode (Handling Long Output)

Lesson 8 of 10 Beginner Track: tmux + Codex CLI

Codex can produce verbose explanations and long code blocks that scroll past your screen. Copy mode is tmux’s built-in feature for scrolling back through output, searching for specific text, and reviewing anything you missed—without asking Codex to repeat itself.

Why Copy Mode Matters

In a normal terminal, once output scrolls off the top of your screen, it’s gone (or buried in a limited scrollback buffer). Tmux’s copy mode gives you full control: you can scroll line by line, jump pages at a time, and search for specific strings—all within the pane’s history.

This is especially useful with Codex, which often produces detailed explanations alongside code. Instead of re-prompting “show me that again,” you scroll back and find it.

Copy Mode Keys

Key Action
Ctrl+J [ Enter copy mode
↑ / ↓ Scroll line by line
Page Up / Page Down Scroll a full page
Ctrl+S Search forward (incremental)
n Jump to next search match
q Exit copy mode

Exercise

1 Enter copy mode

In any pane (the Codex pane is a good choice since it has lots of output), press:

Ctrl+J [

You’ll notice the top-right corner of the pane shows a position indicator (like [0/150]). This means you’re in copy mode and can navigate the scrollback buffer.

2 Scroll through output

Use the arrow keys and page keys to browse Codex’s previous output:

# Scroll line by line ↑ / ↓ # Scroll a full page Page Up / Page Down

Scroll up until you can see some of Codex’s earlier responses from previous lessons.

3 Search for specific text

Instead of scrolling manually, use incremental search. Press Ctrl+S, then type your search term:

Ctrl+S then type: done

Matches highlight as you type. Press Enter to stop at the current match—useful for finding the code block where Codex implemented the done command. Press n to jump to the next match.

4 Exit copy mode

When you’re done reviewing, press q to return to normal mode:

q

You’re back to your regular pane and can type commands as usual.

Tip: Copy mode works in any pane, not just the Codex pane. It’s equally useful for scrolling back through test output, build logs, or git history in your shell pane.

Checkpoint

  • Ctrl+J [ enters copy mode
  • You can scroll with arrow keys and Page Up/Down
  • Ctrl+S searches forward; n jumps to next match
  • q exits copy mode
  • You can review Codex’s previous answers without asking it to repeat itself
