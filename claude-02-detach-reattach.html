Detach & Reattach

Lesson 2 of 10 Beginner Track: tmux + Claude Code

Detaching is tmux’s killer feature. You can disconnect from a running session and reconnect later—everything keeps running in the background as if you never left. This is critical for long-running AI tasks, unstable SSH connections, and switching between projects without losing state.

Why Detaching Matters

Without tmux, closing a terminal kills every process inside it. With tmux, your session is independent of the terminal window. You can:

  • Survive SSH disconnects—your remote work keeps running
  • Let long-running processes finish while you do other things
  • Switch contexts between projects without stopping anything

The pattern is simple: detach to leave a session running, attach to come back to it.

Exercise

1 Confirm you’re in your tmux session

You should still be inside the tutorial session from Lesson 1. Check for the status bar at the bottom of your terminal showing [tutorial].

2 Start a command that shows activity

Run a loop that prints the current time every two seconds. This gives you visible proof that the session stays alive after you detach:

while true; do date; sleep 2; done
3 Detach from the session

Press Ctrl+J d (prefix, then d for “detach”).

You’ll land back at your regular terminal. The tmux status bar is gone—but the session is still running in the background.

4 Verify the session is still alive

List all running tmux sessions:

tmux list-sessions

You should see your tutorial session listed, along with the number of windows and the creation time.

5 Reattach to the session

Connect back to the session by name:

tmux attach -t tutorial

You’re back inside. Notice that the date loop has been printing timestamps the entire time you were away.

6 Stop the date loop

Press Ctrl+C to interrupt the running loop and return to a normal prompt.

Checkpoint

  • After detaching, tmux ls shows your tutorial session still running
  • After reattaching, the date output continued while you were away
  • Your session survived the full detach/attach cycle

Why This Matters

Imagine you’re SSH’d into a server and your internet flickers. Without tmux, your process dies. With tmux, you just reattach—your process never noticed you left.

The detach/attach pattern and AI CLIs: When Claude Code is working on a long task—generating files, running tests, refactoring code—you can detach from the session, close your laptop, and come back later. Claude keeps working. This becomes especially powerful when you start running multiple sessions in parallel later in this track.
