Deterministic Tools Over Prompts

Lesson 4 of 10 Intermediate Track: Guardrails for Speed

You could write “always use 2-space indentation” in your prompt. Or you could run a formatter that enforces it in milliseconds with zero tokens spent. The principle: if a deterministic tool can enforce a rule, don’t ask the LLM to remember it.

The Prompt vs. Tool Decision

For every rule you want an AI agent to follow, ask: can a deterministic tool enforce this? If yes, use the tool. If no, use the prompt. Here is how common rules break down:

Rule Prompt Approach Tool Approach Winner
Code formatting “Use 2-space indent” Prettier / ESLint --fix Tool
Import ordering “Sort imports alphabetically” eslint-plugin-import Tool
No console.log in prod “Remove debug statements” ESLint no-console rule Tool
API naming conventions “Use camelCase for endpoints” Custom linter rule Tool
Don’t modify config files “Never edit .env” Hook (PreToolUse) Tool

The pattern: prompts are suggestions. Tools are enforcement.

The Diff-Only Principle

When you add a linter or formatter to an agentic workflow, don’t run it on the entire codebase—run it only on changed files. This keeps the feedback loop fast and avoids flooding the agent’s context with irrelevant output.

lint-staged.sh — only lint what changed
#!/bin/bash
# lint-staged.sh â€” only lint what changed
STAGED=$(git diff --cached --name-only --diff-filter=ACM | grep -E '\.(ts|tsx|js|jsx)$')
if [ -z "$STAGED" ]; then
  echo "No staged files to lint"
  exit 0
fi
npx eslint $STAGED

Why this matters: speed and token savings. Full-codebase linting wastes time and floods the context with noise the agent has to parse. Diff-only linting gives the agent exactly the signal it needs—errors in the files it just touched.

Wiring Tools into Agent Workflows

There are three places to run deterministic tools in an agentic workflow:

  1. Before the agent starts (pre-check: is the codebase clean?). If the linter already shows errors, the agent will waste tokens trying to understand pre-existing problems.
  2. Inside the agent’s loop (as a hook or as part of the prompt: “after making changes, run ./lint-staged.sh”). The agent sees the output and can fix issues itself without you intervening.
  3. After the agent finishes (post-validation before you review). A final gate that catches anything the agent missed.

The most powerful position is inside the loop. When a deterministic tool runs inside the agent’s edit-test cycle, the agent can self-correct. This is guardrails enabling speed: the agent moves fast because the tool catches mistakes immediately, and the agent fixes them without waiting for you.

Exercise

1 Check for an existing linter

Check if your current project has a linter configured:

npx eslint --version cat .eslintrc*

If you don’t have a linter, you can install one with npm init @eslint/config or use any project that already has one.

2 Create a lint-staged script

If your project has a linter, create a lint-staged.sh script that runs the linter only on staged files. Use the script from the Diff-Only Principle section above. Make it executable:

chmod +x lint-staged.sh
3 Run a headless agent with the linter in the loop

Launch a headless Claude agent that makes a change and then runs the linter:

tmux new-session -d -s lint-test tmux send-keys -t lint-test 'claude -p --dangerously-skip-permissions "Add a helper function to utils.ts, then run ./lint-staged.sh and fix any issues"' Enter
4 Check the results

Attach to the session and review: did the agent self-correct based on linter output?

tmux attach -t lint-test

Look for the pattern: the agent makes a change, the linter reports an issue, and the agent fixes it—all without your intervention. This is the power of deterministic tools inside the agentic loop.

Checkpoint

  • Deterministic tools enforce rules with 100% reliability; prompts are suggestions
  • Run tools on diffs/staged files only, not the entire codebase
  • The most powerful position for a tool is inside the agentic loop
  • Formatting, linting, and style rules should never consume prompt tokens
