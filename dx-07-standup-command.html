The Standup Command

Lesson 7 of 10 Advanced Track: Developer Toolkit

dx standup summarizes your recent git activity into a standup-ready report. It pulls your commits and file-change stats, sends them to an AI for summarization, and outputs a structured update—perfect for async standups or simply remembering what you worked on yesterday.

Git History as AI Context

This command returns to the non-interactive pattern you used for dx review and dx pr: gather data, construct a prompt, run the AI, print the result, exit. The difference is in what gets collected. Instead of staged diffs or commit ranges, dx standup queries your git log filtered by author and time range.

The command collects two pieces of information:

Data Git Command Purpose
Commit summaries git log --oneline --all A concise list of what was done
File change stats git log --stat --all Which files changed and by how much

Both are filtered with --author (your email from git config) and --since (defaulting to "yesterday"). The AI then transforms this raw log into a structured standup format with three sections: what was done, what’s next, and any blockers it can infer.

Customizable time range: Pass --since="3 days ago" or --since="last monday" to adjust the window. Git’s date parsing is flexible—it accepts relative dates, ISO dates, and natural-language expressions.

Exercise: Build the Standup Command

1 Create the standup command

This script queries your git history, builds a structured prompt, and gets an AI-generated standup summary:

~/projects/dx/commands/standup
#!/bin/bash
set -e

SCRIPT_DIR="$(cd "$(dirname "$0")/.." && pwd)"
source "$SCRIPT_DIR/lib.sh"
parse_ai_flag "$@"

# Parse optional --since flag
SINCE="yesterday"
for arg in "$@"; do
  case "$arg" in
    --since=*) SINCE="${arg#--since=}" ;;
  esac
done

# Get git log
AUTHOR=$(git config user.email)
COMMITS=$(git log --author="$AUTHOR" --since="$SINCE" --oneline --all 2>/dev/null || echo "")
DIFF_STAT=$(git log --author="$AUTHOR" --since="$SINCE" --stat --all 2>/dev/null | tail -50 || echo "")

if [[ -z "$COMMITS" ]]; then
  echo "No commits found since $SINCE"
  echo "Try: dx standup --since='3 days ago'"
  exit 0
fi

echo "Generating standup summary with $AI_CLI..."
echo ""

PROMPT="Summarize this git activity for a standup update. Be concise and use bullet points.

Commits since $SINCE:
$COMMITS

Files changed:
$DIFF_STAT

Format:
**Yesterday/Recently:**
- <what was done>

**Today/Next:**
- <logical next steps based on the work>

**Blockers:**
- <any potential issues you can infer, or 'None'>
"

ai_prompt "$PROMPT"

Make it executable:

chmod +x commands/standup

Key details:

  • --since flag parsing — Loops through arguments looking for --since=* and extracts the value with parameter expansion (${arg#--since=}). Defaults to "yesterday" if not provided.
  • git config user.email — Automatically filters the log to only your commits, so the standup reflects your work even in a shared repo.
  • --all — Searches across all branches, catching work on feature branches that hasn’t been merged yet.
  • tail -50 — Trims the diff stats to the last 50 lines to keep the prompt within token limits on busy days.
  • Structured prompt format — The prompt explicitly requests a three-section standup format, so the AI output is ready to paste into Slack or a standup bot.
2 Test with your dx project

You’ve been building commands across several lessons, so there should be commit history to work with. Stage and commit everything first:

cd ~/projects/dx git add -A git commit -m "Add dx toolkit commands"

Now generate a standup. Since the commits might not be from “yesterday,” widen the window:

./dx standup --since="1 week ago"

You should see something like:

Yesterday/Recently: - Added dx toolkit commands including review, pr, explain, test, and standup - Set up project skeleton with dispatcher and shared AI library Today/Next: - Add popup mode for quick AI interactions - Implement file watcher for continuous analysis Blockers: - None

The exact output depends on your git history and which AI you’re using, but it should follow the three-section format.

3 Try different time ranges

Experiment with different --since values to see how the summary changes:

# Just today's work ./dx standup --since="today" # Last three days ./dx standup --since="3 days ago" # Since a specific date ./dx standup --since="2025-01-01"

If no commits are found in the given range, the script exits early with a helpful message suggesting you try a wider window.

Your project structure now looks like this:

~/projects/dx/ ├── dx # Main dispatcher script ├── lib.sh # Shared AI functions └── commands/ ├── review # AI-powered code review ├── pr # AI-generated pull requests ├── explain # Interactive code explainer ├── test # Test runner with AI analysis └── standup # Git activity summarizer

Checkpoint

  • dx standup summarizes your recent git activity into a structured standup report
  • The --since flag lets you adjust the time range (defaults to "yesterday")
  • Output follows a consistent Yesterday / Today / Blockers format ready for sharing
