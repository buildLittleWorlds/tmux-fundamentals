The Supervisor Workflow

Lesson 5 of 10 Beginner Track: tmux + Codex CLI

This is the core pattern for AI-assisted development: Codex codes, you test—side by side. In this lesson you’ll develop a rhythm of prompting, testing, and iterating as you add real features to the td tool.

The Rhythm

The supervisor workflow is a tight feedback loop between you and Codex:

  1. Tell Codex what you want — describe the feature in the right pane
  2. Watch it generate code — you can see the output in real time
  3. Switch panes and test — move left, run the code, check the result
  4. Report back if something’s wrong — switch right, describe the problem
  5. Repeat

This cycle is fast because everything is visible at once. No window switching, no copy-pasting between apps—just a prefix key and an arrow.

What you’ll build: Two new features for td—an add command that stores tasks to a file, and a list command that shows them with line numbers.

Exercise

1 Confirm your two-pane layout

You should have two panes: left for testing, right with Codex running. If not, set this up as you did in Lesson 4.

2 Request the “add” command

In the Codex pane (right), type this prompt:

Add an "add" command to td.sh so I can run: ./td.sh add "Buy groceries" It should store tasks in a file called ~/.td-tasks (one per line).

Watch Codex generate the code. When the output stops, the feature is ready to test.

3 Test the “add” command

Move to the left pane (Ctrl+J ←) and try adding some tasks:

./td.sh add "Buy groceries" ./td.sh add "Call mom" ./td.sh add "Learn tmux" cat ~/.td-tasks

The cat command should show all three tasks, one per line. If something is wrong, switch back to Codex and describe the issue.

4 Request the “list” command

Switch back to the Codex pane (Ctrl+J →) and request the next feature:

Add a "list" command that shows all tasks with line numbers.
5 Test the “list” command

Move to the left pane (Ctrl+J ←) and run:

./td.sh list

You should see your tasks printed with numbers. This is the supervisor workflow in action—prompt, test, iterate.

Checkpoint

  • You can add tasks: ./td.sh add "something"
  • Tasks are stored in ~/.td-tasks
  • You can list tasks: ./td.sh list
  • Output shows numbered tasks

What’s Next: So far you’ve been working inside a single window. In Lesson 6: Windows, you’ll learn to create separate workspaces within your session—like browser tabs for your terminal—so you can keep a dedicated window for git operations.
