Launching the Agents

Lesson 2 of 10 Advanced Track: Multi-Agent Orchestration

With three sessions running, you need a way to start AI agents in the background sessions without leaving your control seat. In this lesson you’ll learn tmux send-keys—the command that lets you type into any session remotely—and tmux capture-pane to peek at what’s happening inside each one.

Remote Control with send-keys

The tmux send-keys command sends keystrokes to a target session as if you were typing them there. The syntax is:

tmux send-keys -t session-name 'command here' Enter

The -t flag targets a specific session. The literal word Enter at the end simulates pressing the Enter key. Without it, the command would appear typed but never execute.

Why this matters: In a multi-agent workflow, you never need to leave the control session. You launch agents, send them prompts, and check their output—all from one place. This is the foundation of the orchestration pattern you’ll use throughout this track.

Reading Output with capture-pane

To see what’s happening in a background session, use tmux capture-pane:

tmux capture-pane -t session-name -p | tail -20

The -p flag prints the captured content to stdout (instead of into a tmux buffer), so you can pipe it through tail to see just the most recent lines. This gives you a read-only snapshot of what the agent is doing.

Exercise: Start Both Agents

1 Launch Claude in the claude-api session

From your control session, send the command to start Claude Code in the background session:

tmux send-keys -t claude-api 'claude' Enter

Claude is now starting up in the claude-api session. You won’t see anything happen in your current terminal—the keystrokes were sent to the other session.

2 Launch Codex in the codex-cli session

Do the same for Codex CLI:

tmux send-keys -t codex-cli 'codex --no-alt-screen' Enter

The --no-alt-screen flag keeps Codex output in the normal terminal buffer, which makes it easier to capture with capture-pane later.

3 Verify both agents are running

Peek at each session to confirm the agents started successfully:

# Quick peek at Claude's session tmux capture-pane -t claude-api -p | tail -20 # Quick peek at Codex's session tmux capture-pane -t codex-cli -p | tail -20

You should see each agent’s startup output—a welcome message, prompt, or loading indicator. If you see just a shell prompt, the agent may still be loading; wait a few seconds and try again.

4 Split your control session for coordination

Set up a two-pane layout in your control session—one side for commands, the other for testing:

# Split the control session vertically Ctrl+J %
  • Left pane — your command center (sending commands to agents)
  • Right pane — for testing and integration work

Switch between panes with Ctrl+J ← and Ctrl+J →.

Command Reference

Command What It Does
tmux send-keys -t name 'cmd' Enter Send a command to a named session
tmux capture-pane -t name -p Print the visible contents of a session’s pane
tmux capture-pane -t name -p | tail -N Show just the last N lines of a session’s output
Ctrl+J % Split the current window into left/right panes

Checkpoint

  • Both AI agents are running in background sessions
  • You can see their output with tmux capture-pane
  • Your control session has a split-pane layout ready for coordination
