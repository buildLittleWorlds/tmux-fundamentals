Capture and Review

Lesson 9 of 10 Advanced Track: Multi-Agent Orchestration

AI agents produce a lot of output—status messages, generated code, error traces, conversational responses. Once a session ends, all of that scrollback vanishes unless you save it. In this lesson you’ll capture full session histories to log files and build a summary script that gives you a quick overview of what each agent accomplished.

Why Capture Session Output

You’ve already used tmux capture-pane for quick peeks at agent output. But that command can do much more—with the -S flag, it captures the entire scrollback buffer, not just what’s currently visible. This gives you a complete record of everything that happened in a session.

Session logs are useful for:

  • Debugging — trace back to where something went wrong
  • Code review — see exactly what the agent generated and in what order
  • Documentation — keep a record of how a feature was built
  • Learning — review the agent’s problem-solving approach after the fact

Exercise: Capture Session Histories

1 Capture Claude’s full session history

Save the entire scrollback buffer from the Claude session to a log file:

tmux capture-pane -t claude-api -p -S -1000 > ~/projects/bookmarks/claude-session.log

The -S -1000 flag tells tmux to start capturing from 1000 lines back in the scrollback buffer. The -p flag sends the output to stdout so you can redirect it to a file.

2 Capture Codex’s full session history

Do the same for the Codex session:

tmux capture-pane -t codex-cli -p -S -1000 > ~/projects/bookmarks/codex-session.log

Now you have two log files—one for each agent—that you can review, search, or diff at any time.

Scrollback depth: The -1000 value captures up to 1000 lines of history. If your agents have been especially chatty, increase it (e.g., -S -5000). The maximum depends on your tmux history-limit setting—the default is 2000 lines.

Exercise: Build a Summary Script

3 Create the summary script

Write a script that reports on what each agent produced:

~/projects/bookmarks/summarize-session.sh
#!/bin/bash
echo "=== Session Summary ==="
echo ""
echo "Files created by Claude (api/):"
find api -type f -name "*.js" -o -name "*.json" 2>/dev/null | head -10
echo ""
echo "Files created by Codex (cli/):"
find cli -type f -name "*.js" -o -name "*.json" 2>/dev/null | head -10
echo ""
echo "Recent Claude activity:"
tail -20 claude-session.log 2>/dev/null | grep -v "^$" | tail -5
echo ""
echo "Recent Codex activity:"
tail -20 codex-session.log 2>/dev/null | grep -v "^$" | tail -5

Make it executable:

chmod +x ~/projects/bookmarks/summarize-session.sh
4 Run the summary

Execute the script from the project root:

cd ~/projects/bookmarks ./summarize-session.sh

You’ll see a quick report showing which files each agent created and their most recent activity. This is a fast way to get your bearings after stepping away from a session or before starting a new round of work.

What the script reports

Section What it shows
Files created by Claude JavaScript and JSON files in api/—the backend artifacts
Files created by Codex JavaScript and JSON files in cli/—the CLI artifacts
Recent Claude activity The last 5 non-empty lines from Claude’s session log
Recent Codex activity The last 5 non-empty lines from Codex’s session log

Checkpoint

  • You can capture full session histories with tmux capture-pane -S -1000
  • You have log files for both agents that persist after the sessions end
  • A summary script gives you a quick overview of both agents’ work
