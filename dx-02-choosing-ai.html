Choosing Your AI

Lesson 2 of 10 Advanced Track: Developer Toolkit

Every dx command needs to talk to an AI, but which one? In this lesson you’ll build lib.sh—a shared library that lets any subcommand work with either Claude or Codex, selected at runtime with a simple --ai flag. This single file will power every command you build for the rest of the track.

AI Abstraction

Rather than hard-coding a specific AI CLI into each subcommand, lib.sh provides a layer of indirection. Every command sources this file, calls parse_ai_flag to read the user’s preference, and then uses one of three helper functions depending on how it needs to interact with the AI.

Function Mode How It Works
ai_prompt Non-interactive Sends a one-shot prompt, gets a response, and exits
ai_interactive Interactive Launches the AI in a tmux session and attaches you to it
ai_capture Capture Reads the AI’s output from a tmux pane using capture-pane

The interactive and capture functions use tmux directly—creating sessions, sending keystrokes, and reading pane buffers. This is where the tmux scripting skills from the beginner tracks pay off.

Exercise: Create the Shared Library

1 Create lib.sh

This file goes in the project root alongside the dx script. Every subcommand will source it:

~/projects/dx/lib.sh
#!/bin/bash
# Shared functions for dx commands

# Parse --ai flag from arguments
parse_ai_flag() {
  AI_CLI="claude"  # default
  AI_PROMPT_FLAG="-p"

  for arg in "$@"; do
    case "$arg" in
      --ai=claude)
        AI_CLI="claude"
        AI_PROMPT_FLAG="-p"
        ;;
      --ai=codex)
        AI_CLI="codex"
        AI_PROMPT_FLAG="-p"
        ;;
    esac
  done

  export AI_CLI AI_PROMPT_FLAG
}

# Run a prompt non-interactively
ai_prompt() {
  local prompt="$1"
  $AI_CLI $AI_PROMPT_FLAG "$prompt"
}

# Run AI interactively in a tmux pane
ai_interactive() {
  local initial_prompt="$1"
  local session_name="${2:-dx-ai}"

  # Create or reuse session
  if ! tmux has-session -t "$session_name" 2>/dev/null; then
    tmux new-session -d -s "$session_name"
  fi

  # Start AI with initial prompt
  tmux send-keys -t "$session_name" "$AI_CLI" Enter
  sleep 2  # Wait for AI to start

  if [[ -n "$initial_prompt" ]]; then
    tmux send-keys -t "$session_name" "$initial_prompt" Enter
  fi

  # Attach to the session
  tmux attach -t "$session_name"
}

# Capture AI response from a session
ai_capture() {
  local session_name="${1:-dx-ai}"
  tmux capture-pane -t "$session_name" -p -S -100
}

Walk through what each function does:

  • parse_ai_flag — Loops through the arguments looking for --ai=claude or --ai=codex. Sets two exported variables: AI_CLI (the command name) and AI_PROMPT_FLAG (the flag for non-interactive prompts). Defaults to Claude.
  • ai_prompt — Runs the selected AI CLI with a single prompt string and exits. Used by commands like dx review that just need a one-shot answer.
  • ai_interactive — Creates a dedicated tmux session, launches the AI inside it, optionally sends an initial prompt, then attaches you. Used when you want a conversation.
  • ai_capture — Uses tmux capture-pane to read the last 100 lines from a session. Useful for grabbing AI output programmatically after a non-interactive run.
2 Test the library

Source the file and verify that the flag parsing works:

source lib.sh parse_ai_flag --ai=codex echo "Using: $AI_CLI with flag: $AI_PROMPT_FLAG"

You should see:

Using: codex with flag: -p

Now test the default (no flag):

parse_ai_flag echo "Using: $AI_CLI with flag: $AI_PROMPT_FLAG"

You should see claude as the default. Your project structure now looks like this:

~/projects/dx/ ├── dx # Main dispatcher script ├── lib.sh # Shared AI functions └── commands/ # Subcommand scripts go here

Checkpoint

  • lib.sh provides shared functions for AI interaction
  • AI can be selected with --ai=claude or --ai=codex
  • Both interactive and non-interactive modes are supported
