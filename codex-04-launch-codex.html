Launch Codex CLI

Lesson 4 of 10 Beginner Track: tmux + Codex CLI

Now that you can split panes, it’s time to put them to work. In this lesson you’ll launch Codex CLI in one pane and start building a real project—a todo CLI called td—while testing the results in the other pane.

The Two-Pane Setup

The layout you’ll use throughout the rest of this track is simple: Codex on the right, your shell on the left. You give Codex instructions in one pane and immediately test the output in the other. No tab switching, no window juggling—everything is visible at once.

What you’ll build: A minimal CLI todo app called td in a single bash file. By the end of this lesson, running ./td.sh --help will show usage information.

Scrollback tip: If Codex’s TUI causes odd scrollback behaviour inside tmux, launch it with the --no-alt-screen flag. This keeps output in the normal terminal buffer where tmux can manage it.

Exercise

1 Start with a single pane

If you still have multiple panes from the previous lesson, close the extras by typing exit in each until you’re down to one pane.

2 Create a project directory

Set up a folder for the td tool:

mkdir -p ~/projects/td && cd ~/projects/td
3 Split into two panes

Create a vertical split so you have a left pane and a right pane:

Ctrl+J %

Your cursor is now in the right pane.

4 Launch Codex in the right pane

Start Codex with the scrollback-friendly flag:

codex --no-alt-screen

Wait for Codex to finish loading and show its prompt.

5 Give Codex its first task

Once Codex is ready, type this prompt:

Create a simple CLI todo app called "td" in a single file called td.sh. For now, just make it show help text when run with --help or no arguments. Keep it minimal - just bash, no dependencies.

Watch as Codex generates the file. When the output stops, the file is ready to test.

6 Test in the left pane

Move to the left pane and verify what Codex created:

Ctrl+J ←

Then make the script executable and run it:

chmod +x td.sh ./td.sh --help

You should see usage information printed to the terminal. If it doesn’t work, switch back to Codex (Ctrl+J →) and describe the problem.

Checkpoint

  • Codex is running in the right pane
  • A file td.sh exists in your ~/projects/td directory
  • Running ./td.sh --help shows usage information

What’s Next: You’ve got Codex generating code and a testing pane to verify it. In Lesson 5: The Supervisor Workflow, you’ll develop a rhythm—prompt Codex, test the result, report back—and add real features to td along the way.
