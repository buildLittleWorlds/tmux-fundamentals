Background Claude Tasks

Lesson 7 of 10 Beginner Track: tmux + Claude Code

Sometimes Claude needs time to work through a larger change—a multi-feature request, a refactor, or a complex bug fix. Rather than watching and waiting, you can background the task and keep using the Claude pane. This lesson teaches you to send Claude’s work into the background, check on its progress, and bring it back when it’s done.

Backgrounding in Claude Code

Claude Code has a built-in backgrounding feature. When Claude is working on a task, you can send that operation to the background and get your prompt back immediately. This lets you ask quick questions, check documentation, or start thinking about your next request—all while Claude continues working behind the scenes.

The key commands are:

Command What It Does
Ctrl+J Background the current Claude operation
/status Check if a background task is still running
/resume Bring the background task back to the foreground

Note: This is a Claude Code feature, not a tmux feature. The Ctrl+J shortcut here is intercepted by Claude Code itself to background its current operation. It works because your tmux prefix requires a second key after Ctrl+J—pressing Ctrl+J alone with no follow-up key is what Claude Code captures.

Exercise

1 Go to your Claude pane

Navigate to window 0 (Ctrl+J 0) and make sure you’re in the Claude pane (right side). Claude Code should be running from earlier lessons.

2 Give Claude a bigger task

Request multiple features at once—this will take Claude a moment to implement:

Add these features to td.sh: 1. A "done" command: ./td.sh done 2 (marks task 2 as complete by removing it) 2. A "clear" command: ./td.sh clear (removes all tasks) 3. Better error handling for invalid commands
3 Background the task

While Claude is working, press:

Ctrl+J

This backgrounds the current Claude operation. Your prompt comes back immediately—Claude is still working, but now you can type.

4 Check the task status

With your prompt back, type:

/status

This shows whether Claude is still working in the background. You can check as many times as you like.

5 Resume the task

When you’re ready to see Claude’s output, bring the task back to the foreground:

/resume

If the task already finished, you’ll see the completed output. If it’s still running, you’ll watch it continue in real time.

6 Test the new features

Once Claude is done, switch to the left pane (Ctrl+J ←) and test all three new features:

./td.sh list ./td.sh done 1 ./td.sh list ./td.sh clear ./td.sh list

The done command should remove a task by number, and clear should remove all tasks. After clearing, the list should be empty.

Checkpoint

  • You can background Claude with Ctrl+J (no follow-up key)
  • /status shows background task progress
  • /resume brings the task back to the foreground
  • ./td.sh done <n> removes a task by number
  • ./td.sh clear removes all tasks

When to Use Backgrounding

Backgrounding is most useful when Claude has enough context to work independently for a while:

  • Long refactors — restructuring code across multiple functions
  • Multi-file changes — adding a feature that touches several files
  • Quick questions — you want to ask Claude something else while a task runs

Keep in mind: Backgrounding works best for self-contained tasks where Claude doesn’t need your input mid-way. If Claude is likely to ask a clarifying question, it’s better to stay in the foreground so you can respond immediately.
